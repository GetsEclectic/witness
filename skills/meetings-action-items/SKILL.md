# meetings-action-items

The witness summaries at `$WITNESS_MEETINGS_DIR/<slug>/summary.md` contain an `## Action items` section with one item per line, formatted:

```
- [ ] <owner>: <what> (due <when>)
```

Aggregate those across meetings and present a living to-do list, filtered to **items the user personally owns** unless they say otherwise.

## How to use this skill

1. **Determine scope.** Unless the user specifies a range, default to the last 14 days of meetings (`$WITNESS_MEETINGS_DIR/<current-month>-*` etc., up through today).

2. **Grep the summaries.**
   ```
   rg -A 20 "^## Action items" "$WITNESS_MEETINGS_DIR"/*/summary.md
   ```
   Parse each `- [ ]` line under that header, capturing:
   - source slug
   - owner (often the user's name, "you", or first-person)
   - action text
   - due date if present

3. **Filter by ownership.** Default to items the user owns (their name, `you`, `me`, first-person). If the user asks "what did <Person> commit to", filter by that name instead.

4. **Sort sensibly.** Items with an explicit due date first (earliest first); then undated items by recency of the source meeting.

5. **Present.** Markdown checklist, grouped by due status:
   - **Overdue / today**
   - **This week**
   - **Later / undated**

   Each line: `- [ ] <action> — from <slug> (due <when>)`.

## What not to do

- **Don't invent action items.** If the summary says nothing, say nothing.
- **Don't dedupe aggressively.** Two meetings that both mention "confirm Q2 headcount" are probably related but not always identical — flag them as possibly the same rather than silently merging.
- **Don't claim something is complete.** The pipeline has no notion of completion. If the user says "this is done", they need to edit `summary.md` (or tell you which file and line to edit). Defer to the user on state changes.

## Edge case

If `summary.md` is missing for a recent meeting, it may still be being generated by the post-processing pipeline or the summarize step failed. Check `witness.log` in that folder and mention the gap — don't silently skip it.
